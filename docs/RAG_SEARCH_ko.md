# RAG 검색

**RAG 검색** 탭은 시맨틱 벡터 검색, 키워드 필터링, 청크 편집, 그리고 결과를 Chat 또는 Discussion으로 전송하기 위한 전용 인터페이스를 제공합니다.

![RAG 검색](images/rag-search.png)

## 검색

1. 드롭다운에서 **RAG 설정**을 선택합니다 (각 설정에는 고유한 인덱스, embedding 모델 및 매개변수가 있습니다)
2. 쿼리를 입력하고 Enter를 누르거나 검색 버튼을 클릭합니다
3. 필요에 따라 **Top K** (최대 결과 수)와 **Score Threshold** (최소 유사도)를 조정합니다

결과는 쿼리 embedding과 각 인덱싱된 청크 간의 코사인 유사도에 따라 순위가 매겨집니다.

## 키워드 필터

시맨틱 검색 후 결과 목록 상단의 키워드 필터로 결과를 좁힐 수 있습니다. 여러 필터 필드를 조합하여 정밀한 필터링이 가능합니다.

![키워드 필터](images/rag-search-keyword.png)

- **필드 내** — 공백으로 구분된 키워드는 **OR** 로직 (하나라도 일치하면 매칭)
- **필드 간** — 여러 필드는 **AND** 로직 (모든 필드의 조건을 충족해야 함)
- **+ AND** 버튼으로 필터 필드 추가
- **✕**로 필터 필드 삭제
- 청크 텍스트와 파일 경로 모두에서 검색
- 텍스트의 공백이 정규화됩니다 (줄바꿈, 전각 스페이스가 축소됨). PDF 추출 아티팩트가 매칭을 방해하지 않도록 합니다
- 공백을 제거한 텍스트 버전에서도 매칭합니다. PDF 추출 시 공백으로 분리된 CJK 단어도 매칭됩니다 (예: "3つのコア機能"을 검색하면 "3 つのコア機能"과 매칭)
- "전체 선택" 체크박스와 카운트는 필터링된 뷰를 반영
- 모든 필터를 지우면 전체 결과가 다시 표시됩니다

### AI 키워드 제안

각 필터 필드에는 AI를 사용하여 키워드를 동의어와 관련어로 확장하는 **✦** 버튼이 있습니다.

- 키워드를 입력한 후 ✦을 클릭
- 설정된 **AI 정리 모델**이 관련어를 생성하고 필드 내용을 대체합니다
- 입력이 영어가 아닌 경우, 영어 번역 및 관련 영어 용어도 포함됩니다
- 원래 키워드를 부분 문자열로 포함하는 중복 용어는 자동으로 제거됩니다 (OR 필터링을 개선하지 않으므로)
- **↩** (실행 취소) 버튼으로 원래 키워드를 복원할 수 있습니다
- **AI 정리 모델** (검색 설정 톱니바퀴 아이콘)에서 모델 선택이 필요합니다

임베딩 유사도 검색에서 놓칠 수 있는 용어 변형을 이미 검색된 결과 내에서 포착하는 데 유용합니다.

## 결과 선택

- 결과 행을 클릭하면 선택 상태가 전환됩니다
- **모두 선택** 체크박스를 사용하여 표시된 (필터링된) 모든 결과를 선택/해제할 수 있습니다
- **선택됨** 개수는 필터링된 뷰뿐만 아니라 전체 결과에서 선택된 수를 표시합니다

## 결과를 Chat 또는 Discussion으로 전송

체크박스로 결과를 선택한 후 버튼 중 하나를 클릭합니다:

- **Chat** — 결과가 Chat 입력 영역에 첨부 파일로 추가됩니다. Chat의 RAG 드롭다운은 중복 컨텍스트 주입을 방지하기 위해 자동으로 "none"으로 설정됩니다.
- **Discussion** — 결과가 Discussion 패널에 첨부 파일로 추가되고 탭이 Discussion으로 전환됩니다.

![결과를 Discussion으로 전송](images/rag-search-discussion.png)

텍스트 결과는 편집 가능한 텍스트 첨부 파일이 됩니다. 미디어 결과 (이미지, PDF, 오디오, 비디오)는 바이너리 파일로 첨부됩니다.

**Chat에서 편집:** Chat으로 결과를 전송한 후, 소스 경로가 있는 텍스트 첨부 파일은 입력 영역에서 클릭할 수 있습니다. 클릭하면 모달이 열려 전송 전에 내용을 검토하고 편집할 수 있습니다.

![Chat에서 RAG 결과 편집](images/rag-search-chat.png)

## 청크 편집

연필 아이콘 (텍스트 결과를 펼쳤을 때 표시됨)을 클릭하면 청크 편집기 모달이 열립니다.

![청크 편집기 모달](images/rag-search-edit.png)

편집기에서 다음을 수행할 수 있습니다:

- **텍스트 편집** — 청크 내용을 자유롭게 수정합니다. 변경 사항은 검색 결과 목록에 다시 저장됩니다.
- **이전 청크 불러오기** — `▲ Load previous chunk`를 클릭하면 같은 파일의 이전 청크가 앞에 추가됩니다. 청크 간의 겹치는 부분은 자동으로 제거됩니다.
- **다음 청크 불러오기** — `▼ Load next chunk`를 클릭하면 같은 파일의 다음 청크가 뒤에 추가됩니다. 겹치는 부분은 자동으로 제거됩니다.
- **결합 후 편집** — 인접 청크를 불러온 후 전체 텍스트를 하나의 블록으로 편집할 수 있습니다. 저장하면 결과가 업데이트됩니다.

시맨틱 검색이 주변 텍스트의 중요한 맥락이 누락된 청크를 반환할 때 유용합니다.

## AI로 다듬기

청크 편집기에서 **✨ Refine with AI**를 클릭하면 LLM을 사용하여 텍스트를 자동으로 확장하고 정리합니다.

**작동 방식:**

1. **초기 확장** — 이전 3개와 다음 3개 청크를 병렬로 로드합니다
2. **AI 평가** — LLM이 검색 쿼리에 대해 텍스트에 충분한 맥락이 있는지 평가합니다. 더 필요한 경우 지정된 방향으로 3개의 청크를 추가 로드합니다 (최대 5회 반복)
3. **다듬기** — LLM이 결합된 텍스트를 정리합니다: 청크 분할 흔적, 끊어진 문장, 노이즈를 제거하면서 의미 있는 정보를 모두 보존합니다. 결과가 편집기로 스트리밍됩니다.

**설정:** 검색 설정(톱니바퀴 아이콘)의 **AI Refine Model** 드롭다운에서 모델을 선택합니다. 모델이 선택되지 않으면 버튼이 비활성화됩니다.

**참고 사항:**
- 사용 후 버튼이 숨겨집니다 (편집 세션당 1회 작업)
- 다듬기 중 및 이후에는 이전/다음 청크 링크가 숨겨집니다
- 처리 중에는 텍스트 영역이 비활성화되어 작업 진행을 표시합니다
- 콘텐츠의 원래 언어가 유지됩니다

## PDF 결과 처리

PDF 검색 결과에는 첨부 모드를 선택할 수 있는 결과별 드롭다운이 있습니다:

- **As text** — PDF.js를 사용하여 PDF에서 텍스트를 추출합니다. 추출된 텍스트는 결과 미리보기에 표시되며, 키워드 필터링을 지원하고, 청크 편집기에서 편집할 수 있습니다. vault 파일과 외부 (절대 경로) PDF 모두에서 작동합니다.
- **As PDF chunk** — 인라인 미리보기가 포함된 원본 PDF 페이지, 바이너리로 첨부

검색 결과가 로드되면 모든 PDF 결과에 대해 백그라운드에서 자동으로 텍스트 추출이 실행되어, 메타데이터 레이블이 실제 콘텐츠로 대체됩니다. 이를 통해 실제 PDF 텍스트에 대한 키워드 필터링과 편집이 가능해집니다.

## 인덱스 설정

검색 바의 톱니바퀴 아이콘을 클릭하면 인라인 인덱스 설정이 열립니다:

- **Chunk Size** — 청크당 문자 수
- **Chunk Overlap** — 인접 청크 간의 문자 겹침 수
- **PDF Chunk Pages** — embedding 청크당 PDF 페이지 수 (1–6)
- **Target Folders** — 인덱싱을 특정 폴더로 제한 (쉼표로 구분)
- **Exclude Patterns** — 파일을 제외하는 정규식 패턴 (한 줄에 하나)
- **Search File Extensions** — 검색을 특정 파일 유형으로 제한 (쉼표로 구분)
- **AI Refine Model** — 청크 편집기의 "Refine with AI"에 사용할 LLM 모델 선택 (없음 = 비활성화)
- **Sync** 버튼 — 진행률 표시줄과 마지막 동기화 타임스탬프 포함
- **인덱싱된 파일** 목록 — 파일별 청크 수 표시

## Chat에서의 RAG vs 검색에서의 RAG

| | Chat + RAG 드롭다운 | 검색 → 선택 → Chat/Discussion |
|---|---|---|
| **컨텍스트 주입** | 시스템 프롬프트 (자동) | 사용자 메시지 첨부 파일 |
| **편집** | 전송 전 편집 불가 | 첨부 파일을 클릭하여 모달에서 편집 |
| **매개변수** | RAG 설정 기본값 사용 | 검색마다 조정 가능 (Top K, 임계값) |
| **결과 선택** | 모든 결과가 자동으로 포함 | 사용자가 포함할 결과를 선택 |
| **인접 청크** | 사용 불가 | 편집기에서 이전/다음 청크 불러오기 |
| **키워드 필터** | 사용 불가 | 선택 전에 결과 필터링 |
| **AI 다듬기** | 사용 불가 | 청크를 자동 확장하고 LLM으로 다듬기 |

검색 흐름은 LLM에 전송되는 컨텍스트를 더 세밀하게 제어할 수 있게 해줍니다. Chat의 RAG 드롭다운은 완전 자동 컨텍스트 주입을 위한 편리한 단축 기능입니다.

## Discussion에서의 RAG

Discussion 패널은 두 가지 방법으로 RAG를 지원합니다:

1. **검색 → Discussion** — 검색 탭에서 결과를 선택하고 Discussion 버튼을 클릭합니다. 결과가 첨부 파일로 추가되며 시작 전에 편집할 수 있습니다.
2. **RAG 드롭다운** — Discussion 패널에서 직접 RAG 설정을 선택합니다. 주제 텍스트가 검색 쿼리로 사용됩니다. 이미 첨부 파일이 있는 경우 (검색 또는 파일 업로드로부터) 이 옵션은 비활성화됩니다.

RAG 컨텍스트와 첨부 파일은 중복 API 호출을 방지하기 위해 토론의 **첫 번째 턴**에서만 전송됩니다. 이후 턴은 이미 RAG 컨텍스트가 반영된 토론 기록을 기반으로 진행됩니다.
